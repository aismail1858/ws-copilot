# user-research

Bedarfsanalyse, Nutzerinterviews, Personas, User Journeys, Feedback-Auswertungen.

## Inhalt

- Nutzerinterviews und Beobachtungen
- Persona-Definitionen und Segmentierung
- User Journey Maps und Pain Points
- Feedback-Zusammenfassungen (Support, Umfragen, Beta-Tests)
- Priorisierung von Nutzerbedürfnissen

## Hinweise

- Befragte/Teilnehmer nur anonymisiert oder mit Einverständnis dokumentieren.
- Verknüpfungen zu User Stories (`docs/user-stories-and-journeys.md`) und Task-MDs pflegen.
