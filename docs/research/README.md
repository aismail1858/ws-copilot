# research

Zentraler Ordner für alle Forschungs- und Recherche-Arbeiten zum Projekt.

Unterordner sind nach Forschungsbereich gegliedert. Jeder Unterordner enthält eine eigene README.md und, falls vorhanden, `.mmd`-Diagramme für visuelle Darstellungen der Ergebnisse.

## Struktur

```
research/
├── README.md
├── market-research/      ← Marktanalyse, Wettbewerb, Trends
├── ui-research/          ← UI/UX-Vergleiche, Patterns, Usability
├── tech-research/        ← Technologie-Evaluation, Vergleiche, Prototyping
├── user-research/        ← Bedarfsanalyse, Interviews, Personas
├── regulatory-research/  ← Compliance, Datenschutz, regulatorische Anforderungen
└── ...                   ← weitere Bereiche nach Bedarf
```

## Verbindliche Regel

- Recherche-Ergebnisse werden nur in den dafür vorgesehenen Unterordnern abgelegt.
- Jede neue Forschungs-Doku wird mit einer Referenz auf die zugehörige Task-MD (`docs/tasks/TASK-*.md`) versehen.
- Bei Abschluss einer Task mit Forschungsanteil wird die relevante Doku hier angelegt oder aktualisiert — Pflicht lt. `AGENTS.md` Section 4.
- Diagramme zu Forschungsergebnissen werden als `.mmd`-Dateien gespeichert.
